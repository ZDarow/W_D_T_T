## 2. Архитектурные паттерны

### 2.1 Multi-Interface Architecture (AmneziaWG)

**Описание:** Каждый уникальный набор CPS-параметров создаёт отдельный `awgN`-интерфейс. Клиенты с одинаковыми параметрами разделяют интерфейс. Интерфейсы создаются по требованию и удаляются когда последний клиент отключается.

**Применение в WDTT:**
- Вместо одного WireGuard TUN на всех клиентов — создать отдельный интерфейс на группу клиентов с одинаковым профилем обфускации
- **Приоритет: средний** (после рефакторинга монолита)

### 2.2 Connection Pool Pattern (awg-server)

**Описание:** Пул WireGuard-интерфейсов с автоматическим менеджментом жизненного цикла. Каждый интерфейс слушает на своём UDP-порту.

**Применение в WDTT:**
- Порт 56001 сейчас один для всех. Переход на `port = base_port + idx` позволит разнести трафик
- **Приоритет: низкий** (требует изменений в Android-клиенте)

### 2.3 Clean Architecture / Packet разделение (swgp-go)

**Описание:** swgp-go использует пакетную структуру: `conn/`, `packet/`, `service/`, `cmd/`, `internal/`. Каждый пакет — чёткая ответственность.

**Применение в WDTT:** Прямой шаблон для разделения 2138-строчного `server.go`:
```
cmd/wdtt/main.go     — точка входа, DI
internal/crypto/     — obfsWrapPacket, obfsUnwrapPacket, getAEAD, salamanderXOR
internal/db/         — Database, PasswordEntry, save/load
internal/bot/        — Telegram handlers
internal/wg/         — WireGuard management
internal/proxy/      — handleConn, authClient, proxyToWG, proxyToClient
internal/nat/        — iptables/nft управление
```
**Приоритет: высокий** (Sprint 2 архитектуры)

### 2.4 Adapter Pattern (sing-box)

**Описание:** sing-box использует интерфейсы `adapter.Service`, `adapter.Inbound`, `adapter.Outbound` для абстракции протоколов. Это позволяет добавлять новые протоколы без изменения ядра.

**Применение в WDTT:**
- Определить интерфейс `Transport` для разных способов обфускации (RTP, Salamander, AmneziaWG-CPS)
- **Приоритет: средний** (после разделения на пакеты)

## Итого

**18 источников найдено.** Ключевые инсайты:

1. **AmneziaWG** (⭐ 1.7k) — самый зрелый форк WG с обфускацией. CPS-подход отличается от RTP WDTT, но может быть скомбинирован
2. **awg-server** (⭐ 21) — эталонная архитектура Go-сервера для VPN. Прямой референс для разделения монолита WDTT
3. **swgp-go** (⭐ 354) — replay protection через timestamp. Решает C-SEC-04
4. **sing-box** (⭐ 35.4k) — зрелая модульная архитектура. Adapter pattern
5. **go-iptables** — замена exec.Command для iptables (утечка правил при shutdown)
6. **nDPI wireguard.c** — точные сигнатуры обнаружения WG. Валидация RTP-маскировки
7. **net4people/#88** — 200+ комментариев с полевым опытом обхода DPI
