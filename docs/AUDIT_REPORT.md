## 3. Технический долг

### 3.1 Критический (необходимо исправить)

| # | Проблема | Файл:строка | Риск |
|---|----------|-------------|------|
| **TD-1** | `net.ResolveUDPAddr` и `selfsign.GenerateSelfSigned` — ошибки игнорируются | server.go:1433 | Сервер может запуститься с некорректным адресом или без сертификата → паника при Accept |
| **TD-2** | `setupFullConeNAT` всегда возвращает `nil` (никогда не ошибку) | server.go:1193 | Вызывающий код не может определить неудачу настройки NAT |
| **TD-3** | SIGTERM: `os.Exit(0)` через 2 сек — не дожидается завершения `handleConn` | server.go:1412 | При перезапуске теряются активные соединения, возможна потеря данных |
| **TD-4** | `aeadCache` — AEAD-объекты кешируются бесконечно, утечка при ротации ключей | server.go:1723 | При частой смене паролей растёт память |
| **TD-5** | `aeadCache.Delete(string(entry.key))` — строка-ключ остаётся в памяти после zeroBytes | server.go:275 | Ключи не полностью очищаются из RAM |

### 3.2 Высокий

| # | Проблема | Файл:строка | Риск |
|---|----------|-------------|------|
| **TD-6** | `os.WriteFile("/proc/sys/net/ipv4/ip_forward", ...)` — ошибки игнорируются | server.go:1213 | IP forwarding может не включиться → NAT не работает |
| **TD-7** | `exec.Command(...).Run()` в `setupFullConeNAT` — ошибки игнорируются | server.go:1217-1219 | Правила iptables могут не установиться |
| **TD-8** | `clientConn.Write([]byte(...))` — ошибки игнорируются во многих местах | server.go:1567, 1583, 1602 | Клиент не узнает об ошибке |
| **TD-9** | `botLoop` (410 строк) — функция слишком большая | server.go:407 | Сложно поддерживать, тестировать и ревьюить |
| **TD-10** | `handleConn` (240 строк) — функция слишком большая | server.go:1478 | Высокая цикломатическая сложность |

### 3.3 Средний

| # | Проблема | Файл:строка | Риск |
|---|----------|-------------|------|
| **TD-11** | go.mod содержит неиспользуемые зависимости: `pion/turn/v5`, `connutil`, `uuid` | go.mod | Увеличенный размер бинарника, лишние транзитивные зависимости |
| **TD-12** | `generatePassword()` — слабый fallback при отказе `rand.Read` (time-based) | server.go:155 | Предсказуемые пароли в редких условиях |
| **TD-13** | `runCmdSilent` — все ошибки игнорируются | server.go:1092 | Невозможно диагностировать проблемы |
| **TD-14** | Нет rate-limiting на аутентификацию | server.go:1478 | Уязвимость к brute-force атакам |
| **TD-15** | Нет лимита на количество одновременных соединений | server.go:1456 | Уязвимость к ресурсному истощению |
| **TD-16** | `obfsBuildNonce` — пустые байты 6-7 (padding) в nonce | server.go:1776 | Не использует все 12 байт nonce, снижает entropy |
| **TD-17** | Нет go.sum в проекте wdtt (скопирован с VPS) | server/go.sum | Не гарантируется воспроизводимость сборки |

## 4. Уязвимости безопасности

### 4.1 Актуальные

| # | Уязвимость | Серьёзность | Описание |
|---|-----------|-------------|----------|
| **SEC-1** | Нет лимита на попытки аутентификации | Высокая | Brute-force паролей через DTLS-соединения |
| **SEC-2** | Нет лимита на параллельные соединения | Высокая | DoS через исчерпание goroutines/дескрипторов |
| **SEC-3** | Отсутствует проверка целостности конфигурации | Средняя | Подмена config.json → доступ к серверу |
| **SEC-4** | Пароль передаётся в открытом виде в первом пакете DTLS | Средняя | (защищено DTLS) Потенциально уязвимо до завершения handshake |
| **SEC-5** | WG-ключи хранятся в файле `wg-keys.dat` mode 0600 | Средняя | Физический доступ к серверу → компрометация всех клиентов |
| **SEC-6** | `BotToken` и `AdminID` в открытом виде в JSON | Средняя | Физический доступ к серверу → управление через Telegram |
| **SEC-7** | Отсутствует защита от повторного воспроизведения (replay) | Средняя | Повторная отправка перехваченных DTLS-пакетов |

### 4.2 Смягчённые

| # | Уязвимость | Защита |
|---|-----------|--------|
| **SEC-8** | Перехват трафика | DTLS 1.2 + AEAD (ChaCha20-Poly1305) |
| **SEC-9** | Анализ содержимого пакетов | RTP-обфускация + опциональный Salamander-XOR |
| **SEC-10** | Timing analysis | Опциональный jitter (1-10ms) |
| **SEC-11** | Анализ размеров пакетов | Случайный padding (1-255 байт) |
